# Step 1: Requirements Gathering Instructions

**Purpose:** Collect initial requirements from all available sources (stakeholder notes, sketches, images).

**Input files:**
- `input.md` (context, stakeholder notes, etc.)
- `input_images/` (optional UI sketches to be translated into text requirements)
- See `example/input.md` for a sample input for this step.

**Output files:**
- `output.yaml` (all extracted, unprocessed requirements)
- See `example/output.yaml` for a sample output for this step.

- Fill `input_sources.md` with all relevant context and sources.
- (Optional) Place UI sketches in the `input_images/` subfolder.
	- Extract and write all raw requirements in `output_raw_requirements.yaml`.
- See [../instructions.md](../instructions.md) for the full pipeline overview.
